# منصة راصد الوطنية: أدلة المستخدم والمساهمين

يقدم هذا المستند أدلة شاملة لكل من المستخدمين النهائيين والمساهمين في منصة راصد الوطنية. ويغطي وظائف المنصة من منظور المستخدم ويقدم إرشادات مفصلة للمطورين الذين يرغبون في المساهمة في المشروع.

## 1. أدلة المستخدم

يوجز هذا القسم الميزات والوظائف المختلفة المتاحة لأدوار المستخدمين المختلفة داخل منصة راصد الوطنية.

### 1.1. أدوار المستخدمين والأذونات

تستخدم المنصة نظام التحكم في الوصول المستند إلى الأدوار (RBAC) لإدارة أذونات المستخدمين. يتم تحديد الأدوار التالية:

| الدور | الوصف |
| --- | --- |
| **المسؤول** | لديه حق الوصول الكامل إلى جميع ميزات المنصة، بما في ذلك إدارة المستخدمين وإعدادات النظام وإدارة المحتوى. |
| **مشرف المحتوى** | يمكنه مراجعة المحتوى المقدم من المحررين والمستخدمين الآخرين والموافقة عليه. |
| **المحرر** | يمكنه إنشاء المحتوى وتحريره وتقديمه للمراجعة والنشر. |
| **المشاهد** | لديه حق الوصول للقراءة فقط إلى المحتوى العام على المنصة. |
| **المستخدم** | مستخدم مسجل عام لديه حق الوصول إلى الميزات الأساسية مثل التعليق ووضع الإشارات المرجعية ولوحات المعلومات المخصصة. |
| **المشرف** | يمكنه إدارة المحتوى الذي ينشئه المستخدمون مثل التعليقات وإدخالات سجل الزوار. |
| **العضو** | دور مستخدم خاص لديه حق الوصول إلى المحتوى والخدمات الحصرية للأعضاء فقط. |

### 1.2. الميزات الأساسية للمنصة

#### 1.2.1. استهلاك المحتوى

- **الأخبار:** ابق على اطلاع بآخر المقالات والإعلانات.
- **مقاطع الفيديو:** شاهد مجموعة متنوعة من محتوى الفيديو.
- **معرض الصور:** تصفح ألبومات الصور.
- **الأدلة:** الوصول إلى الأدلة الإرشادية والمعلوماتية.
- **العروض التقديمية:** عرض وتنزيل العروض التقديمية.
- **الروابط:** اكتشف قائمة منسقة من الروابط المفيدة.

#### 1.2.2. الميزات التفاعلية

- **التعليقات:** شارك في المناقشات حول المقالات الإخبارية والمحتويات الأخرى.
- **الإعجابات:** أظهر تقديرك للمحتوى.
- **الإشارات المرجعية:** احفظ المحتوى لمشاهدته لاحقًا.
- **سجل الزوار:** اترك رسائل وملاحظات لمسؤولي المنصة.

#### 1.2.3. التخصيص

- **لوحة التحكم الخاصة بي:** لوحة تحكم مخصصة تحتوي على معلومات ذات صلة ووصول سريع إلى الميزات.
- **الملف الشخصي:** إدارة ملف تعريف المستخدم الخاص بك وتفضيلاتك.
- **الإشعارات:** تلقي إشعارات في الوقت الفعلي للأحداث المهمة.

### 1.3. الميزات الإدارية

يتمتع المسؤولون والأدوار الأخرى ذات الامتيازات بالوصول إلى لوحة تحكم قوية لإدارة المنصة:

- **إدارة المستخدمين:** إدارة حسابات المستخدمين وأدوارهم وأذوناتهم.
- **إدارة المحتوى:** إنشاء المحتوى وتحريره والموافقة عليه ونشره.
- **ترتيب المحتوى:** تنظيم ترتيب عرض الأخبار ومقاطع الفيديو.
- **الجدولة:** جدولة المحتوى للنشر في المستقبل.
- **التحليلات:** عرض تحليلات مفصلة وإنشاء تقارير حول استخدام المنصة.
- **الإعدادات:** تكوين إعدادات النظام المختلفة، بما في ذلك SMTP لإشعارات البريد الإلكتروني.

## 2. أدلة المساهمين

هذا القسم مخصص للمطورين والمساهمين الذين يرغبون في المساعدة في بناء وتحسين منصة راصد الوطنية.

### 2.1. البدء

#### 2.1.1. المتطلبات الأساسية

- **Node.js:** مطلوب لكل من تطوير الواجهة الأمامية والخلفية.
- **Git:** للتحكم في الإصدار.

#### 2.1.2. التثبيت

1.  **استنساخ المستودع:**

    ```bash
    git clone https://github.com/alramady/alqasem.git
    cd alqasem
    ```

2.  **تثبيت الاعتماديات:**

    المشروع عبارة عن مستودع أحادي. لتثبيت الاعتماديات لكل من العميل والخادم، ستحتاج إلى الانتقال إلى الدلائل المعنية وتشغيل أمر تثبيت مدير الحزم.

### 2.2. سير عمل التطوير

#### 2.2.1. الواجهة الأمامية

الواجهة الأمامية هي تطبيق React مبني باستخدام Vite و TypeScript. ويستخدم TailwindCSS للتصميم و Radix UI لمكونات واجهة المستخدم.

- **تشغيل خادم التطوير:**

  ```bash
  cd client
  pnpm dev
  ```

#### 2.2.2. الواجهة الخلفية

الواجهة الخلفية هي تطبيق Node.js يستخدم tRPC لاتصال واجهة برمجة التطبيقات و Drizzle ORM للتفاعل مع قاعدة البيانات.

- **تشغيل خادم التطوير:**

  ```bash
  cd server
  pnpm dev
  ```

### 2.3. معايير الترميز

- **TypeScript:** يجب كتابة كل التعليمات البرمجية بلغة TypeScript مع فحص صارم للنوع.
- **التدقيق والتنسيق:** يستخدم المشروع ESLint و Prettier لفرض نمط كود متسق. يرجى التأكد من أن التعليمات البرمجية الخاصة بك تلتزم بهذه المعايير قبل إرسال طلب سحب.

### 2.4. تقديم المساهمات

1.  **تفرع المستودع.**
2.  **إنشاء فرع جديد لميزتك أو إصلاح الخلل.**
3.  **قم بإجراء تغييراتك وقم بتثبيتها برسائل واضحة ووصفية.**
4.  **ادفع تغييراتك إلى تفرعك.**
5.  **افتح طلب سحب إلى المستودع الرئيسي.**

ستتم مراجعة مساهمتك من قبل مشرفي المشروع، وبمجرد الموافقة عليها، سيتم دمجها في الفرع الرئيسي.
